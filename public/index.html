About This Project

An educational exploration of web analytics data collection methods

🎯 Purpose

This website was built as an educational tool to demonstrate and compare three major methods of collecting visitor data on the web. By navigating through these pages, you are generating real data that can be viewed in the Dashboard.

🏗️ Technology Stack

  • Server: Node.js + Express
  • Logging: Morgan (Apache format)
  • Analytics: Google Analytics 4 (gtag.js)
  • Frontend: Vanilla HTML, CSS, JS

📚 How It Works

Every page visit, click, and interaction is captured simultaneously by server-side logs, the Google Analytics tag, and is visible through browser developer tools — allowing a direct comparison of the data each method collects.

🔬 Academic Context

Web analytics is essential for understanding user behavior, optimizing website performance, and making data-driven decisions. This project provides hands-on experience with the three most common data collection approaches used by web professionals.

🏷️ Google Analytics Page Tag Code

<!-- Google Analytics (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){ dataLayer.push(arguments); }
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX', {
    page_title: document.title,
    send_page_view: true
  });

  // Custom event tracking
  function trackEvent(category, action, label) {
    gtag('event', action, {
      event_category: category,
      event_label: label
    });
  }
</script>

Browser Developer Tools Data

Open Developer Tools

Press F12 or Cmd+Option+I (Mac) / Ctrl+Shift+I (Windows) to open Browser Developer Tools.

2

Network Tab

Navigate to the Network tab to see all HTTP requests, response headers, payloads, timing, and waterfall charts.

3

Application Tab

Check Application → Cookies and Local Storage to see stored tracking data including GA cookies.

4

View JavaScript logs, errors, and run commands like document.cookie or navigator.userAgent.

5

Record and analyze page load performance, paint timings, and JavaScript execution times.

🗄️ Server Logs

✅ Strengths

  • No client-side code required
  • Captures ALL requests (including bots)
  • Full IP address visibility
  • No privacy consent needed
  • Works even if JS is disabled

❌ Limitations

  • No user interaction data
  • Cannot track cached pages
  • Difficult to identify unique users
  • Large storage requirements
  • Requires manual analysis tools

📈 Google Analytics

✅ Strengths

  • Rich user behavior insights
  • Real-time dashboard
  • Demographic & geographic data
  • Conversion & goal tracking
  • Cross-device tracking

❌ Limitations

  • Requires JavaScript enabled
  • Blocked by ad blockers
  • Data sent to third party (Google)
  • GDPR/privacy compliance needed
  • Sampling on large datasets

🛠️ Browser Dev Tools

✅ Strengths

  • Deepest level of detail
  • Real-time inspection
  • Full network request visibility
  • Performance profiling
  • No setup or external tools needed

❌ Limitations

  • Only inspects a single session
  • Data not stored or aggregated
  • Requires technical knowledge
  • Cannot be automated easily
  • Not suitable for production monitoring
